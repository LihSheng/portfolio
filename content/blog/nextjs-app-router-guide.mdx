# Next.js App Router: A Complete Guide

The App Router is a new paradigm in Next.js that brings server components, improved routing, and better performance to React applications.

## What is the App Router?

The App Router is built on React's latest features and provides:

- **Server Components** by default
- **Improved routing** with file-system based routing
- **Better performance** with automatic code splitting
- **Enhanced developer experience** with TypeScript support

## Key Concepts

### Server vs Client Components

By default, components in the `app` directory are Server Components:

```tsx
// app/page.tsx - Server Component (default)
export default function HomePage() {
  return (
    <div>
      <h1>Welcome to my site</h1>
      <p>This renders on the server!</p>
    </div>
  );
}
```

Use the `"use client"` directive for client-side interactivity:

```tsx
"use client";

import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  );
}
```

### Layouts and Templates

Create shared layouts with `layout.tsx`:

```tsx
// app/layout.tsx
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <nav>Navigation</nav>
        <main>{children}</main>
        <footer>Footer</footer>
      </body>
    </html>
  );
}
```

### Dynamic Routes

Create dynamic routes with square brackets:

```tsx
// app/blog/[slug]/page.tsx
interface PageProps {
  params: { slug: string };
}

export default function BlogPost({ params }: PageProps) {
  return <h1>Post: {params.slug}</h1>;
}

export async function generateStaticParams() {
  const posts = await getPosts();
  
  return posts.map((post) => ({
    slug: post.slug,
  }));
}
```

## Data Fetching

### Server Components

Fetch data directly in Server Components:

```tsx
async function getPosts() {
  const res = await fetch('https://api.example.com/posts');
  return res.json();
}

export default async function BlogPage() {
  const posts = await getPosts();
  
  return (
    <div>
      {posts.map((post) => (
        <article key={post.id}>
          <h2>{post.title}</h2>
          <p>{post.excerpt}</p>
        </article>
      ))}
    </div>
  );
}
```

### Loading and Error States

Handle loading and errors with special files:

```tsx
// app/blog/loading.tsx
export default function Loading() {
  return <div>Loading posts...</div>;
}

// app/blog/error.tsx
'use client';

export default function Error({
  error,
  reset,
}: {
  error: Error;
  reset: () => void;
}) {
  return (
    <div>
      <h2>Something went wrong!</h2>
      <button onClick={() => reset()}>Try again</button>
    </div>
  );
}
```

## Route Handlers (API Routes)

Create API endpoints with `route.ts`:

```tsx
// app/api/posts/route.ts
import { NextRequest, NextResponse } from 'next/server';

export async function GET(request: NextRequest) {
  const posts = await getPosts();
  return NextResponse.json(posts);
}

export async function POST(request: NextRequest) {
  const body = await request.json();
  const newPost = await createPost(body);
  return NextResponse.json(newPost, { status: 201 });
}
```

## Metadata and SEO

Generate metadata for better SEO:

```tsx
import type { Metadata } from 'next';

export const metadata: Metadata = {
  title: 'My Blog',
  description: 'A blog about web development',
};

// Dynamic metadata
export async function generateMetadata({ params }): Promise<Metadata> {
  const post = await getPost(params.slug);
  
  return {
    title: post.title,
    description: post.excerpt,
  };
}
```

## Migration Tips

### From Pages Router

1. **Move files**: `pages/` â†’ `app/`
2. **Update imports**: Use new conventions
3. **Convert to Server Components**: Remove unnecessary client-side code
4. **Update API routes**: Use new `route.ts` format

### Best Practices

- Start with Server Components
- Use Client Components sparingly
- Leverage streaming and Suspense
- Implement proper error boundaries
- Optimize for Core Web Vitals
