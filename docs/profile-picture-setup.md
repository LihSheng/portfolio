# Profile Picture Configuration Guide

## Overview

This guide walks you through setting up and configuring profile pictures for your portfolio website. The profile picture system provides optimized images with fallback mechanisms and accessibility features.

## Quick Setup

### 1. Add Your Profile Picture

Place your profile picture in the `public/images/` directory:

```
public/
  images/
    profile.jpg          # Your main profile picture
    profile-fallback.jpg # Optional fallback image
```

For dynamic profile picture configuration, you can use environment variables:

```bash
# .env.local
NEXT_PUBLIC_PROFILE_PICTURE_URL=/images/profile.jpg
```

Then update your site config:

```typescript
export const siteConfig: SiteConfig = {
  author: {
```

Your profile picture should now appear in:
- Hero section on the homepage

## Detailed Configuration

### Profile Picture Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `profilePicture` | string | No | Path to your main profile picture |
| `profilePictureAlt` | string | No | Alt text for accessibility |
| `avatar` | string | No | SVG fallback (keep for compatibility) |

### Image Requirements

**Recommended specifications:**
- **Format**: JPG, PNG, or WebP
- **Dimensions**: 400x400px minimum (square aspect ratio)
- **File size**: Under 500KB for optimal loading
- **Quality**: High resolution for crisp display on all devices

**Supported formats:**
- JPEG (.jpg, .jpeg)
- PNG (.png)
- WebP (.webp)
- AVIF (.avif) - automatically generated by Next.js

### File Organization

```
public/
  images/
    profile.jpg           # Main profile picture
    profile-hq.jpg        # High-quality version (optional)
    profile-mobile.jpg    # Mobile-optimized version (optional)
    avatars/              # Alternative avatars (optional)
      avatar-1.svg
      avatar-2.svg
```

## Advanced Configuration

### Custom Fallback Strategy

You can customize the fallback behavior by modifying the ProfilePicture component or creating custom utilities:

```typescript
// lib/profile-utils.ts
export const getProfilePictureConfig = () => {
  return {
    primary: siteConfig.author.profilePicture,
    fallback: siteConfig.author.avatar,
    initials: generateInitials(siteConfig.author.name),
    alt: siteConfig.author.profilePictureAlt || `Profile picture of ${siteConfig.author.name}`,
  };
};
```

### Multiple Profile Pictures

For different contexts, you can configure multiple profile pictures:

```typescript
export const siteConfig: SiteConfig = {
  author: {
    name: "Your Name",
    profilePicture: "/images/profile-professional.jpg",
    profilePictureAlt: "Professional headshot of Your Name",
    // Additional profile pictures for different contexts
    casualProfilePicture: "/images/profile-casual.jpg",
    speakingProfilePicture: "/images/profile-speaking.jpg",
  },
};
```

### Remote Images

To use images from external sources, configure Next.js image domains:

```javascript
// next.config.mjs
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'your-cdn-domain.com',
        port: '',
        pathname: '/images/**',
      },
    ],
  },
};
```

Then update your site config:

```typescript
export const siteConfig: SiteConfig = {
  author: {
    profilePicture: "https://your-cdn-domain.com/images/profile.jpg",
    profilePictureAlt: "Professional headshot",
  },
};
```

## Optimization Tips

### Image Optimization

1. **Compress images** before adding to your project
2. **Use appropriate dimensions** (400x400px recommended)
3. **Choose the right format** (WebP for modern browsers, JPG for compatibility)
4. **Optimize for mobile** with responsive sizing

### Performance Considerations

1. **Use priority loading** for above-the-fold images
2. **Implement proper caching** (handled automatically by Next.js)
3. **Monitor loading performance** with browser dev tools
4. **Test on slow connections** to ensure good UX

### SEO Benefits

1. **Descriptive alt text** improves accessibility and SEO
2. **Proper image sizing** prevents layout shift
3. **Optimized formats** improve page load speeds
4. **Structured data** can include profile picture URLs

## Troubleshooting

### Common Issues

**Profile picture not showing:**
```bash
# Check if file exists
ls public/images/profile.jpg

# Verify file permissions
chmod 644 public/images/profile.jpg
```

**Image appears blurry:**
- Use higher resolution source image (800x800px minimum)
- Check image compression settings
- Verify Next.js image optimization is working

**Slow loading:**
- Compress image file size
- Use WebP format
- Enable priority loading for above-the-fold images

**Accessibility warnings:**
- Add descriptive alt text
- Ensure proper contrast ratios
- Test with screen readers

## Migration Guide

### From Avatar-Only Setup

If you're currently using only the `avatar` field:

1. **Keep existing avatar** for backward compatibility
2. **Add profilePicture field** with your actual photo
3. **Add profilePictureAlt** for accessibility
4. **Test both hero and about sections**

### From Custom Implementation

If you have a custom profile picture implementation:

1. **Replace custom components** with ProfilePicture component
2. **Move configuration** to site-config.ts
3. **Update styling** using className prop
4. **Test fallback behavior**
